# Workflow Rules

Conventions for how to work, regardless of what is being produced. Files in this directory describe collaboration, process, and discipline, distinct from artefact-side rules (`code/` for code, `writing/` for text).

---

## Files

| File | Covers |
|---|---|
| `agent.md` | Agent operating modes and collaboration styles |
| `science-project.md` | Science project lifecycle: starting, running, and archiving a mimir project |
| `git.md` | Commit format, branch naming, PR conventions |

## Subdirectories

| Directory | Covers |
|---|---|
| `notes/` | Note-taking systems: rules, projects, codebases, HPC, meetings, machines |
| `quokka/` | Quokka-specific workflow: build, run, MHD, testing, diagnostics |
| `remote-work/` | Working on remote machines: SSH, HPC job workflow, data transfer |
